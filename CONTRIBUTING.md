# Contributing

## Setup

```bash
npm install
npm run build
```

## Guidelines

- Keep packages generic and reusable
- Do not add secrets, private URLs, or organization-specific credentials
- Prefer configuration over hardcoded behavior
- Add or update README examples when behavior changes

## Pull Requests

- Keep changes scoped to one package when possible
- Include tests for behavior changes if the package has tests
- Update documentation for public APIs
